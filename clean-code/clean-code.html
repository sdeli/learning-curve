PROGRAMMING JAVASCRIPT APPLICATIONS - ERIC ELLIOT CLEAN CODE

    CLEAN CODE

  1. Collective agreements on things, how we do something for code consistency
    2. VARIABLES
  2. No magic values
  3. Descriptive variable names - should consist of 2 to 4 words and 8 to 20 characters
  4. Booleans should be stored in variables, with a descriptive name
  5. Use Typescript to show that which variable is what
    6. FUNCTIONS
  6. DOT/Single Responsibility functions Easier to give a descriptive name Easier to write tests for
  7. Pure functions Deterministic easier to write test
  8. Few params as possible, maximum 3 | params passed as keys of an object
  9. More tiny functions are better then few large better for nodes performance optimization
  10. No nested functions => makes logic more difficult to understand
    11. CONTROL FLOW
  11. if conditons should be stored in variable with descriptive name
  12. if statements should have more lines then 3 | content of stament should be outsources into functions with a descriptive name
  13. loops should be wrapped into functions with descriptive names
    14. COMMENTS
  14. Comments are always the sign that you couldn`t express yourself in code
  15. You shouldnt describe a code in comment which is likely to change
    16. ORGANIZATION OF LOGIC
  16. 1 file 1 set of logic if there are more sets of logic (a class called car, which can accelarate, stop, turn on music ....) these should be delegated into separate files and then attached in the main file on the main entity
  17. Logic should be readable from top to the bottom => function definitons should be composed in top-down narrative
  18. No nested logic => No nested functions if there is a logic which needs nesting, then that one should be delegated into separate file as a library or submodule
    19. ARCHITECTURE
  19. mvc pattern
  20. Favor declarative programming over imperative programming => Top Down Analysis
  21. Modules and Classes should have a clean interface interface:Should be documented, what do they need to work properly:
  22. dependencies, parameters
  23. how they need to be used in code
  24. other conditions: they need to have internet connection
