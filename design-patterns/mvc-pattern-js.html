MODEL / Wiev / CONTROLLER

    MODEL
  • Models represent knowledge. A model could be a single object (rather uninteresting), or it could be some structure of objects.
  • Responsible for GETTING and MANIPULATING DATA
  • Interacts with database
  • Communcates wtih controller
  • controller can request data from the model
  • can be where the data lives
    WIEV
  • A view is a (visual) representation of its model
  • the interface of the application
  • communicates with the controller
  • Template Engine
    CONTROLLER
  • A controller is the link between a user and the system.
  • takes in USER INPUT from wiev
  • like process requests: GET, POST, PUT, DELETE
  • mediates between model and wiev
