    DIRECTIVES

  • Specific configuration options, which get set in the config file
  • name value pairlike: server_name mydomain.com
  • A directive can be a set for a given context
  • Global directives can be declared in the global context
    • Standard Directives
    • Array Directives
    • Action Directives
      • can be declared multiple times without overriding previous
      • gets inherited straight down
      • child context can override inheritance by re-declaring directive
    • listen

      The listen directive typically defines which IP address and port that the server block will respond to.

      A block with no listen directive uses the value 0.0.0.0:80.

      A block set to an IP address 111.111.111.111 with no port becomes 111.111.111.111:80

      A block set to port 8888 with no IP address becomes 0.0.0.0:8888

      Nginx checks for the longest matching location blocks

      The first regular expression location that matches the request URI is immediately selected to serve the request.
