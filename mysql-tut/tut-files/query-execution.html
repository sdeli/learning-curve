EXECUTION OF A QUERY

  • the SQL DATABASE constist of 2 major parts
  • STORAGE ENGINE
  • QUERY PROCESSOR
  • STORAGE ENGINE is responsible for reading data from disk into the memory/buffer pool
  • QUERY PROCESSORs job is to:
  • accept all queries submitted to sql server
  • devise a plan to carry out query => QUERY OPTIMIZER => translate the WHAT to HOW
  • execute the plan => EXECUTION ENGINE
  • QUERY PROCESSOR consists of QUERY OPTIMIZER and EXECUTION ENGINE. query optiomizer finds out and passes the plan to the execution engine
  • EXECUTION ENGINE gets the plan, executes and delivers back the results
  • query optimizes has DATABASE CATALOG which has metadata about the db like column names
  • QUERY OPTIMIZER created plan is a program, this program is built up by code generator or from precompiled modules
  • FROM
  • it pulls all the tables into the memory
  • joins are happening here - joins are bound to the from statment
  • SQL ROW SOURCE GENERATION
  • WHERE
  • where fiters the rows based on its statement
  • GROUP BY
  • groups the data remained after the where clause and creates a group for them. the groups are represented by a row. to each group there is a list assigned which includes the members of the groups. this list is only accessible for aggregate functions
  • AGGREGATE FUNCITONS
  • an aggregate function performs a calculation on a set of values and returns a single value (they are ignoring null values)
  • aggregate functions can access the data created by GROUP BY
  • executed here no matter where they are syntactically
  • HAVING
  • in having you can access already aggregation functions
  • SELECT
  • select makes the resulted data accessible
  • DISTINCT
  • ORDER BY
  • LIMIT
    select last_name, count(*) from actors group by last_name;
